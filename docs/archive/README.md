# Archive

Pre-harness era artifacts archived during the post-harness context overhaul (2026-04-13). These files are preserved for historical reference but are not active contracts.

Distilled knowledge now lives in:
- `.agent/rules/` (enforced operational rules)
- `.agent/rules/lessons-learned.md` (bug symptom index)
- `docs/reference/agent-lessons-details.md` (detailed lesson narratives)

## When to read archived docs

- **Investigating a historical decision**: why a wave was shaped a certain way, what acceptance criteria looked like
- **Tracing a UI pattern origin**: cerb-era design briefs show the prototype intent behind shipped components
- **Debugging a legacy contract**: if active cerb spec is unclear, the archived execution brief for that wave may have the original rationale

Do NOT read archived docs to:
- Derive new backend behavior or pipeline contracts
- Treat UI choreography descriptions as implementation requirements
- Copy patterns from old execution briefs into new work
