## §4 系统通知 (Snackbar)

> **理念**: 透明的系统状态更新，不阻断用户流程。
> **行为**: 底部出现（在键盘/导航栏上方），4秒后自动消失。

| 通知类型 | 触发条件 | 内容格式 |
|----------|----------|----------|
| **需要澄清** | 实体有歧义 (低置信度) | `⚠️ 需要澄清: [Entity Name] (歧义请求)` |
| **发现冲突** | 检测到日程重叠 | `⚠️ 发现冲突: [Event A] vs [Event B]` |
| **发现关联** | 后台资料库命中 | `✨ 发现关联: [Entity Name] (置信度: [Score]%)` |
| **客户信息更新** | 通过关联库丰富画像 | `👤 客户信息更新: [Name] (新增偏好/职位...)` |
| **知识库扩充** | 从上下文中学习新事实 | `📚 知识库扩充: [Topic] (来源: [Source])` |
| **关系变动** | 检测到实体连接 | `🔗 关系网变动: [Person A] -> [Person B]` |
| **记忆刷新** | 用户画像/上下文更新 | `🧠 记忆已更新: [Key Information]` |
| **听悟完成** | 后台转写完成 | `📝 转写完成: [File Name] (点击查看)` |
| **任务已创建** | 语音指令处理完毕 | `✅ 日程已创建: [Task Name] [Time]` |

**视觉风格:**
*   **冲突/警告**: 琥珀色/黄色强调。
*   **关联/信息**: Prism 蓝/紫色强调。
*   **动作**: 点击 Snackbar 跳转到相关卡片或上下文。
