## 1. Agent 活动横幅 (Agent Activity Banner)

> **状态**: ✅ 已发布
> **代码**: `AgentActivityBanner.kt`
> **概述**: 使用 **双层结构** 来全面展示 Agent 的活动。这是产品的核心差异化点。

### 结构

| 层级 | 角色 | 示例 |
|------|------|------|
| **Phase (主)** | 高层任务 (总是可见) | "📝 规划分析步骤", "⚙️ 执行工具: PDF生成" |
| **Action (副)** | 具体操作 (可选) | "🧠 思考中...", "📚 检索记忆..." |
| **Trace (迹)** | 流式内容 (可选) | 原生 CoT, 转写, 记忆命中 |

### 视觉状态

1.  **Header (灰, Shimmer - 总是可见)**:
    `思考中... 正在规划分析步骤 [∨]`
2.  **Expanded (≤3 行, 灰, 无 Shimmer)**:
    当 Trace 开始流式传输时自动展开。超过3行自动折叠。
3.  **Collapsed (仅 Header)**:
    用户点击 `[∨]` 展开全部，点击 `[∧]` 折叠。

### 折叠/展开行为
- **初始**: 折叠。
- **Trace 开始**: 自动展开。
- **Trace > 3行**: 自动折叠。

### 设计原则
Banner 绑定的是 **流式活动的存在**，而非 **模式**。
Coach 模式自然跳过思考，Analyst 模式自然展示思考。

## 2. 连接弹窗 (Connectivity Modal)

> **状态**: ✅ 已发布
> **概述**: 点击 Header 中的 `[📶]` 图标触发。管理 SmartBadge 连接状态。

### 状态机

#### State A: 已连接 (Live)
- 显示: `[🟢 3D Badge (Pulse)]`
- 信息: ID, 电量
- 动作: `[⚡ 断开连接]` `[🔄 检查更新]`

#### State B: 已断开 (Offline)
- 显示: `[⚪ Badge (灰度)]`
- 信息: 🔴 离线
- 动作: `[连接设备]` (主按钮)

#### State C: 固件更新
- 显示: `[⬇️ Icon]` 发现新版本 v1.3
- 动作: `[立即同步]`

#### State D: WiFi 不匹配
- 触发: Badge WiFi != Phone WiFi。
- 显示: `[⚠️ Alert]` 网络环境变更。
- 输入: SSID / 密码。
- 动作: `[更新配置]` `[忽略]`

## 3. Knot FAB (悬浮球)

> **状态**: 🔲 延期至 V2
> **概述**: "活体智能" — 一个会呼吸、旋转的悬浮球，提供环境反馈。

### 状态
- **Idle**: 呼吸动画。
- **Thinking**: 旋转 (Analyst 处理中)。
- **Tip Shown**: 点击切换上下文提示气泡。

> **注意**: 已批准但优先级较低，V1 骨架暂不实现。
