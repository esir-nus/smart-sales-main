## 3. Deviation & Resolution Log
* **Attempt 1 Failure**: Total failure across T1-T3 for OS RAM Assembly and Intent Routing. T4 failed at Presentation Layer.
  - **Root Cause**: 
    1. Lightning Router classification thresholds are too strict (`VAGUE` vs `QA`).
    2. Session Working Set is dropping Entity Pointers between turns (`entityKnowledge=0`).
    3. `CRM_TASK` intents are being hard-routed to the Scheduler prompt environment.
  - **Resolution Strategy**:
    * Per discussion with Senior Engineer & Product Owner, **Option 1 will be adopted**: The Natural Language (`CRM_TASK` / Chat) entry point for physical Scheduling will be decoupled. Users will be required to tap physical UI buttons for scheduling to prevent namespace collision and intent overload on the Analyst pipeline.
