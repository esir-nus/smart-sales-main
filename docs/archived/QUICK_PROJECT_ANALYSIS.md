- **产品**: AI 驱动的销售助手 Android 应用

## 6. 技术债务与风险

### 🔴 高风险
1. **Gradle Wrapper 下载**: 受限网络环境无法自动下载，需手动缓存
2. **真实硬件验证不足**: BLE/WiFi 设备未接入常规回归流程
3. **凭据管理**: DashScope/Tingwu/OSS 凭据在 `local.properties`，易误提交

### 🟡 中风险
1. **测试覆盖率**: 28% 覆盖率，关键路径需补充测试
2. **MediaServer baseUrl**: 需手动输入，未与 Connectivity 自动探测打通
3. **播放控制**: MediaPlayer 未处理生命周期/AudioFocus

### 🟢 低风险
1. **文档同步**: 需确保模块 README 与 current-state.md 同步
2. **日志规范**: 需统一使用既有 Tag，避免随意新增

## 9. 改进建议（优先级排序）

### 🔴 高优先级
1. **提升测试覆盖率**: 当前测试代码 8,715 行（31.9%），文件覆盖率 28%，目标提升至 50%+，重点补充集成测试
2. **真实硬件验证**: 接入 BLE/WiFi 设备到常规回归流程
3. **凭据管理优化**: 考虑使用 Android Keystore 或环境变量

### 🟡 中优先级
1. **MediaServer 自动探测**: 与 Connectivity 模块打通，自动获取 baseUrl
2. **播放控制完善**: 处理 MediaPlayer 生命周期和 AudioFocus
3. **监控与指标**: 为 AI 服务添加流量治理和性能指标

### 🟢 低优先级
1. **工具类扩展**: 补充日志/时间/文件工具到 core:util
2. **Fake 实现扩展**: 为测试提供更多 Fake 实现
3. **文档同步自动化**: 建立文档同步检查机制
