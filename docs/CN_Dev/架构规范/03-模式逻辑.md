## 1. 教练模式 (Chat Mode / The Coach)

轻量级对话路径，带有可选的记忆搜索。

### 1.1 流程图 (ASCII Visualization)
```
User Input
    │
    ▼
[Context Builder: Session History]
    │
    ├─▶ LLM Check A: Suggest Analyst? ──(Yes)──▶ Prompt User
    │
    ├─▶ LLM Check B: Memory Threshold Score
    │       │
    │       ├─▶ Score > T: Search Relevancy Library ("Searching memory...")
    │       │
    │       └─▶ Score ≤ T: Session Context Only
    │
    ▼
[Qwen Turbo] ──stream──▶ [ChatPublisher] ──▶ UI
```

### 1.2 上下文策略
1.  **第一步**: 会话历史 (Session History) 为主上下文。
2.  **第二步**: 提示词检查 (Prompt Checks):
    - **Analyst 建议**: 检测是否需要深度分析。需用户确认。
    - **记忆阈值**: 检测是否需要历史上下文 (0-1 分数)。如果 > 阈值，自动搜索 Relevancy Library。

## 2. 分析师模式 (Analyze Mode / The Analyst)

重型推理路径，包含思维轨迹 (Thinking Trace) 和规划表 (Planner Table)。

### 2.1 流程图
```
User Input
    │
    ▼
[Context Builder: Relevancy + Hot Zone]
    │
    ▼
[Qwen Max (Thinking)] ──stream──▶ [Thinking Trace UI]
    │
    ▼
[Qwen Max (Structured)] ──stream──▶ [Planner Table]
    │                                  │
    │                            [Tool Execution]
    │                                  │
    └─async─▶ [Memory Writer] ──▶ Hot Zone
```

### 2.2 上下文策略
1.  查询 **Relevancy Library** (实体上下文, 决策历史)。
2.  查询 **Hot Zone** (最近相关条目)。
3.  生成 `ExecutionPlan` (Plan-Once 模型)。

## 3. 日程模式 (Schedule Mode / Global Top Drawer)

结构化指令路径，具有多步上下文。

### 3.1 流程图
```
Voice Note (ESP32)
    │
    ▼
[Context Builder]
    ├─▶ Step 1: Check Inspirations + Tasks (Today)
    ├─▶ Step 2: Query Relevancy Library (Entity Match)
    └─▶ Step 3: Apply 14-day Rule (Hot vs Cement)
    │
    ▼
[Qwen Structured] ──json──▶ [SchedulePublisher]
                                  │
                            [Calendar API]
                            [Tips Renderer]
```

### 3.2 14天规则 (14-Day Rule)
- 如果 `lastMentionedAt` < 14 天 → 检查 Hot Zone。
- 如果 `lastMentionedAt` ≥ 14 天 → 检查 Cement Zone。

## 4. 模型路由 (Model Router)

基于 **任务类型** 而非模式进行路由。

| 输入条件 | 模型 | 用途 |
|----------|------|------|
| 图片/视频 | `qwen-vl-plus` | 视觉分析 |
| 工具调用 | `qwen3-max` | 32k Tokens |
| 默认 (对话) | `qwen-plus` | 1M Tokens |

**记忆层路由**:
- **RELEVANCY**: `qwen3-max` (精确工具调用)。
- **HOT**: `qwen-plus` (快速导航)。
- **CEMENT**: `qwen-long` (10M Tokens 深层检索)。

## 5. 冲突解决 (Rethink Model)

冲突被视为 **创造性的提示 (Creative Prompts)**，而不是简单的合并决策。

**流程**: 检测 → 建议 → 解决

1.  **检测**: 下午 2:00 重叠。
2.  **建议 (Rethink)**: Agent 提出 "第三种选择" (例如：合并为联合会议)。
3.  **用户选择**:
    - **Override**: 覆盖冲突项。
    - **Merge**: 合并两者。
    - **Rethink**: 用户输入自定义解决方案 → 记录到决策日志 (Decision Log)。
