Removed / Simplified Branches

1. Removed stale WifiDisconnectedReason values:
   - HTTP_UNREACHABLE
   - PHONE_WIFI_UNAVAILABLE
   - PHONE_WIFI_SSID_UNREADABLE
   - NO_KNOWN_CREDENTIAL_FOR_PHONE_WIFI

   Files:
   - app-core/src/main/java/com/smartsales/prism/data/connectivity/legacy/ConnectionModels.kt
   - app-core/src/main/java/com/smartsales/prism/data/connectivity/RealConnectivityService.kt
   - app-core/src/main/java/com/smartsales/prism/data/connectivity/legacy/setup/DeviceSetupViewModel.kt
   - app-core/src/test/java/com/smartsales/prism/data/connectivity/RealConnectivityServiceTest.kt

   Justification:
   - Core Flow says badge Wi-Fi recovery must not depend on reading phone SSID and must branch from BLE state, badge-reported IP/SSID, active HTTP media readiness, and saved user-confirmed credentials.
   - Production grep showed these four enum values were not produced by runtime reconnect/repair code after Sprint 06. The only production references were service/setup display mappings. Production producers now emit:
     - BADGE_WIFI_OFFLINE for IP#0.0.0.0 / unusable badge IP.
     - BADGE_PHONE_NETWORK_MISMATCH for manual repair target-SSID mismatch.
     - CREDENTIAL_REPLAY_FAILED for bounded saved-credential replay exhaustion.
   - Tests using the removed enum values were synthetic coverage for impossible branches, so keeping them made the data flow look larger and more phone-SSID-driven than current code.

   Evidence commands:
   - Before removal:
     - `rg -n "WifiDisconnectedReason\\.(PHONE_WIFI_UNAVAILABLE|PHONE_WIFI_SSID_UNREADABLE|NO_KNOWN_CREDENTIAL_FOR_PHONE_WIFI|HTTP_UNREACHABLE)|WifiDisconnectedReason \\{|enum class WifiDisconnectedReason" app-core/src/main/java/com/smartsales/prism/data/connectivity app-core/src/test/java/com/smartsales/prism/data/connectivity/RealConnectivityServiceTest.kt`
     - 19 references to those stale reasons across enum, service/setup mapping, and tests.
   - After removal:
     - `rg -n "PHONE_WIFI_UNAVAILABLE|PHONE_WIFI_SSID_UNREADABLE|NO_KNOWN_CREDENTIAL_FOR_PHONE_WIFI|HTTP_UNREACHABLE" app-core/src/main/java app-core/src/test/java docs/projects/badge-session-lifecycle/sprints/07-connectivity-dataflow-simplification.md || true`
     - Remaining hits are only `KNOWN_HTTP_UNREACHABLE`, a SIM audio auto-sync latch name, not the removed connectivity reason.

Phone Wi-Fi / SSID Classification
Command:
`rg -n "currentPhoneSsid|phoneSsid|PhoneWifi|PhoneWifiSnapshot|currentNormalizedSsid" app-core/src/main/java/com/smartsales/prism/data/connectivity app-core/src/main/java/com/smartsales/prism/data/audio/SimAudioRepositorySyncSupport.kt app-core/src/main/java/com/smartsales/prism/ui/components/connectivity/ConnectivityViewModel.kt app-core/src/test/java/com/smartsales/prism/data/connectivity app-core/src/test/java/com/smartsales/prism/data/audio app-core/src/test/java/com/smartsales/prism/ui/components/connectivity`

Remaining production uses:
- Prompt prefill:
  - ConnectivityViewModel consumes `ReconnectResult.WifiMismatch.currentPhoneSsid` to prefill the repair form.
  - RealConnectivityService passes `error.phoneSsid` only for the remaining mismatch/offline/replay repair form path.
  - SimAudioRepositorySyncSupport calls `currentNormalizedSsid()` only before manual `promptWifiMismatch`.
- Isolation prompt gating/prefill:
  - SimAudioRepositorySyncSupport and IsolationProbeHelper read `PhoneWifiSnapshot.Connected.isValidated` to classify suspected AP isolation after HTTP/media failure and pass `normalizedSsid` as editable prompt prefill.
- Diagnostic:
  - RealConnectivityBridge logs `phoneSsidForDiag=${result.data.phoneWifiName}` during endpoint resolution.
  - PhoneWifiProvider and FakePhoneWifiProvider are provider/fake infrastructure for the above prompt/diagnostic uses.
- Removable candidate not changed in this sprint:
  - `ConnectivityError.WifiDisconnected.phoneSsid` and `ReconnectResult.WifiMismatch.currentPhoneSsid` are now poorly named because the remaining live producer for BADGE_PHONE_NETWORK_MISMATCH uses the user-submitted target SSID. Renaming those public/domain-facing fields would require a wider interface sync, so Sprint 07 did not alter that API.
