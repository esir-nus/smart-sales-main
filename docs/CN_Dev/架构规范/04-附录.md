## 1. 崩溃恢复与检查点 (Crash Recovery)

**哲学**: 最小检查点策略。只缓存 `EnhancedContext` (昂贵的构建产物)。其他要么已持久化 (聊天记录)，要么构建成本低。

### 1.1 检查点策略
```
CKPT 0: 用户输入 (ChatHistory)
   ↓
CKPT 1: Enhanced Context (ContextCache)
   ↓
CKPT 2: AI 响应 (ChatHistory)
```

**恢复行为**:
- **崩溃于 CKPT 1 之前**: 从头重建上下文 (昂贵)。
- **崩溃于 CKPT 1 之后**: 加载缓存的上下文，恢复 LLM 调用 (跳过昂贵的上下文构建)。

**Context Cache 卫生**:
- **TTL**: 24小时。
- **Scope**: 每个活跃轮次一个检查点。

## 2. 可测试性合约 (Lattice Compliance)

Prism 继承并扩展了 **Lattice 架构** 原则。
- **Prism**: 关注智能 Agent 模式 (记忆, RAG, 流式)。
- **Lattice**: 关注模块化和黑盒隔离。

### 2.1 必需接口与 Fake
每个组件 (Box) 必须有一个接口和一个用于测试的 Fake。

| 接口 | Fake | 用途 |
|------|------|------|
| `LlmProvider` | `FakeLlmProvider` | 返回预设响应 |
| `ContextCache` | `FakeContextCache` | 内存 Map |
| `MemoryRepository` | `FakeMemoryRepository` | 内存存储 |

### 2.2 依赖规则
1.  **禁止 Box-to-Box 导入**: 所有通信必须通过 Orchestrator。
2.  **Domain 层无 Android**: 纯 Kotlin (KMP-ready)。
3.  **Orchestrator 是很薄的**: 委托给 Box，无内联业务逻辑。

## 3. 扩展指南 (Extensions)

1.  **新模式**: 实现 `ModeStrategy`, `LlmProvider`, `ModePublisher`。在 Orchestrator 中注册。
2.  **新工具**: 实现 `ToolAgent` 接口。
3.  **多平台**: 保持 Orchestrator 和 Context Builder 为纯 Kotlin。`MemoryRepository` 是 Android Room 与 iOS CoreData 的分界线。

## 4. 混合 RAG 策略 (Hybrid RAG)

> **[未来规划]**

| 层级 | 范围 | 处理 |
|------|------|------|
| **Tier 1** | 会话记忆 | 100% 端侧 (元数据, 关键词索引) |
| **Tier 2** | 用户文档 | 混合 (小文档本地, 大文档云端 Bailian) |
| **Tier 3** | 全局语义 | 端侧量化 Embeddings (NPU 加速) |
