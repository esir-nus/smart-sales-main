# 记忆系统 (Memory System)

> **原文链接**: [Architecture.md](../../specs/Architecture.md) §5, Appendix B

## 1. 记忆三位一体 (Memory Trinity)

Prism 的记忆系统建立在三个核心存储之上：

```
┌─────────────────────────────────────────────────────────────────┐
│                    PRISM MEMORY TRINITY                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────┐    ┌─────────────────┐                   │
│   │    HOT ZONE     │───▶│  CEMENT ZONE    │                   │
│   │   (活跃区)      │    │  (归档区)       │                   │
│   └────────┬────────┘    └─────────────────┘                   │
│            │                                                    │
│            │ write-through                                      │
│            ▼                                                    │
│   ┌─────────────────────────────────────────┐                  │
│   │         RELEVANCY LIBRARY               │                  │
│   │        (关联库 - 客户索引)               │                  │
│   └─────────────────────────────────────────┘                  │
│                                                                 │
│   + USER PROFILE (静态偏好)                                     │
│   + USER HABIT (学习到的模式)                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

| 存储 | 角色 | 内容 |
|------|------|------|
| **Hot Zone** | 活跃工作 | 今日条目，进行中的会话 (`isArchived=false`) |
| **Cement Zone** | 归档 | 完成超过 14 天的条目 (`isArchived=true`) |
| **Relevancy Library** | 客户索引 | 聚合的实体、别名、决策历史 |
| **User Profile** | 静态设置 | 经验水平、行业、角色 |
| **User Habit** | 习得模式 | 会议时间偏好、客户语调等 |

> **注意**: 旧的 "M1/M2/M3/M4" 术语已**在大脑中移除**。请使用上述直观名称。

---

## 2. 双区模型 (Two-Zone Model)

**Hot Zone 定义**:
- 所有 **活跃条目** (`isArchived = false`)。
- 所有 14 天内 (过去或未来) 的 **日程项**。

**Cement Zone 定义**:
- `isArchived = true` 且日期 > 14 天前。

> **机制**: 相同 Schema，不同标志位。条目通过后台压缩从 Hot 老化进入 Cement。

---

## 3. Relevancy Library (关联库)

持久化、直写的索引，用于 O(1) 实体查找和冲突检测。

### 3.1 Schema
```kotlin
data class RelevancyEntry(
    val entityId: String,           // "z-001" (Person)
    val entityType: String,         // PERSON, PRODUCT, LOCATION, EVENT
    val displayName: String,        // "张伟"
    val aliasesJson: String,        // ["张总", "老张"]
    val decisionLogJson: String,    // 决策历史 (用于 Rethink 学习)
    val metricsHistoryJson: String  // 自定义指标的时间序列
)
```

### 3.2 Metrics History (数据可视化)
存储关键销售指标的历史变更。
- **类型**: 货币 (分), 数量, 时长, 枚举, 日期。
- **用途**: Analyst 模式可直接根据此数据生成趋势图 ("显示张总过去6个月的预算趋势")。

### 3.3 字段更新策略
- **displayName**: 首次写入为准。
- **aliasesJson**: 追加 (去重)。
- **metricsHistoryJson**: **追加** (时间序列)。
- **decisionLogJson**: **追加** (历史)。

## 4. 实体消歧 (Entity Disambiguation)

当语音输入包含歧义引用 (如 "张总" 匹配多人) 时，系统使用 **基于计数的强化学习**。

### 4.1 别名映射
```kotlin
data class AliasMapping(
    val alias: String,              // "张总"
    val entityId: String,           // "z-001"
    val confirmationCount: Int,     // 确认次数 (强化信号)
    val lastConfirmedAt: Long       // 最近时间
)
```

### 4.2 评分算法
1.  **confirmationCount** (40%): 确认次数。
2.  **recencyDecay** (30%): 时间衰减。
3.  **contextMatch** (30%): 当前会话上下文匹配度。

如果 Top1 得分 > 0.85 且显著高于 Top2 → **自动消歧** (带 Disclosure)。
否则 → **显示选择器**。

## 5. 结构化输出模型

LLM 输出分为两个不同部分 — 无内联标记混合。

```
[STRUCTURED SECTION] (JSON)
{ "entities": [...] }

[USER CONTENT] (Clean Text)
「张总希望在周三...」
```

**处理流程**: LLM 输出 → Linter 验证 JSON → Publisher 显示文本 → Writer 写入 JSON 到库。

## 6. 用户画像 (User Profile)

用户级配置 (`UserCenter`)。
- **字段**: 经验水平, 行业, 角色, 语言偏好。
- **用途**: 调整 Coaching 深度和建议的专业度。

---

## 7. 用户习惯 (User Habit)

自动学习的行为模式。

**学习原则**:
1.  **观察阈值**: 3次一致观察后创建。
2.  **信心衰减**: 30天无观察则衰减。
3.  **显式优先**: 用户手动设置的永远不会衰减。
4.  **负向信号**: 用户拒绝建议 → 信心大幅下降。

**示例**: 
- `meeting_time`: `morning` (早上)
- `verbosity`: `bullet_points` (要点式)
