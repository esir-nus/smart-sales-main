## 🏗️ 交互图谱 (TAR 模型)

每个元素定义遵循此严格逻辑：
- **视觉状态 (Visual)**: 用户交互*前*看到什么。
- **触发器 (Trigger)**: 确切的用户输入 (事件)。
- **动画 (Animation)**: 物理效果或过渡曲线 (反馈)。
- **结果 (Result)**: 逻辑状态变更或导航 (结果)。
- **不变量 (Invariant)**: 绝不可打破的硬性规则 (护栏)。
- **状态 (Status)**: 实现状态 (`✅ 已验证`, `🚧 进行中`, `❌ 待定`)。

### 6.2 Agent 活动横幅 (双层结构)

> **设计原则**: Agent 活动使用 **双层层级** 来丰富展示 Agent 的思考。这是核心差异化点。

> **位置**: AgentActivityBanner 渲染 **在聊天记录中内联** 并作为 Agent 认知的永久记录 **持久化**。当 Agent 完成思考，回复气泡作为 **独立、后续的消息** 出现（而不是替换）。这创造了可见的轨迹：用户 → 思考框 → AI 回复。

#### 视觉结构

```
┌─────────────────────────────────────────────────────────┐
│  Phase: 📝 规划分析步骤                                  │  ← 主层 (高层任务)
│  Action: 🧠 思考中...                                    │  ← 次层 (具体操作)
├─────────────────────────────────────────────────────────┤
│  [思考轨迹 / 转写 / 记忆命中...]                          │  ← 主体 (流式内容)
└─────────────────────────────────────────────────────────┘
```

#### 阶段类型 (Phase)

| 阶段 | 图标 | 文案 | 时机 |
|------|------|------|------|
| `PLANNING` | 📝 | "规划分析步骤" / "生成执行计划" | 分析师模式计划生成 |
| `EXECUTING` | ⚙️ | "执行工具: {toolName}" | 运行工具 (图表, 报告等) |
| `RESPONDING` | 💬 | "生成回复" | 标准回复生成 |
| `ERROR` | ⚠️ | "发生错误" | 错误状态 |

#### 动作类型 (Action)

| 动作 | 图标 | 文案 | 轨迹来源 |
|------|------|------|----------|
| `THINKING` | 🧠 | "思考中..." | Qwen3-Max 原生 CoT 流 |
| `PARSING` | 📄 | "解析中... ({filename})" | Qwen-VL 视觉输出 / OCR |
| `TRANSCRIBING` | 🎙️ | "转写中... ({filename})" | 听悟转写流 |
| `RETRIEVING` | 📚 | "检索记忆..." | 关联库命中 |
| `ASSEMBLING` | 📋 | "整理上下文..." | 组装上下文预览 |
| `STREAMING` | ✨ | "生成中..." | LLM 回复流 |

#### 轨迹来源 (原生 vs 合成)

| 来源 | 类型 | 详情 |
|------|------|------|
| **Qwen3-Max CoT** | 原生 | `enable_thinking=true` → `reasoning_content` (分析师: 完整) |
| **Qwen-Plus CoT** | 原生 | `enable_thinking=true` → `reasoning_content` (教练: 截断3行) |
| **Qwen-VL CoT** | 原生 | `enable_thinking=true` → `reasoning_content` (视觉: OCR 轨迹) |
| **听悟** | 原生 | 实时转写 (伪思考) |
| **关联库** | 合成 | 显示命中的实体/记忆 |
| **上下文组装** | 合成 | 显示组装的来源 |

#### 组件签名

```kotlin
data class AgentActivity(
    val phase: ActivityPhase,        // 必需: PLANNING, EXECUTING, 等
    val action: ActivityAction?,     // 可选: THINKING, PARSING, 等
    val trace: String? = null        // 可选: 流式内容
)

@Composable
fun AgentActivityBanner(
    phase: ActivityPhase,            // 必需: 总是可见
    action: ActivityAction?,         // 可选: 如存在则可见
    trace: List<String>?,            // 可选: 主体内容
    modifier: Modifier = Modifier
)
```

#### 渲染规则

| 场景 | 视觉输出 |
|------|----------|
| `action = null, trace = null` | 仅阶段 (如 "⚠️ 网络连接失败") |
| `action = THINKING, trace = null` | 阶段 + 动作, 无主体 |
| `action = THINKING, trace = [...]` | 阶段 + 动作 + 可滚动轨迹 |
